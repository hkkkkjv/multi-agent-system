# StudyFlow — Контекст системы

## Что такое StudyFlow

StudyFlow — мультиагентная система для управления обучением по нескольким темам одновременно.
Помогает студентам не терять фокус, соблюдать дедлайны и понимать сложные концепции.

## Агенты системы

### Supervisor (qwen2.5:7b)
Оркестратор. Получает запрос пользователя и решает кому делегировать:
- Запрос про время/дедлайны → Planner
- Запрос про понимание темы → Tutor
- Оба аспекта → оба агента последовательно

### Planner (qwen2.5:3b)
Специализируется на расписаниях и приоритизации.
Принципы работы:
- Дедлайн завтра = первый приоритет
- Блоки по 45 минут (Pomodoro)
- Перерыв 15 минут каждые 2 блока
- Максимум 4 часа подряд без длинного перерыва

### Tutor (Kimi K2.5)
Объяснение концепций. Принципы:
- Начинает с базового уровня
- Использует аналогии и примеры из жизни
- Находит связи между разными предметами
- Адаптируется под уровень пользователя

### Evaluator (qwen2.5:1.5b)
Контроль качества. Блокирует ответы если:
- Ответ не по теме запроса
- Нет конкретики (пустые слова)
- Есть явные противоречия
При score < 0.8 — retry (максимум 2 попытки)

## Формат взаимодействия

Пользователь пишет запрос в свободной форме на русском языке.
Система возвращает ответ в виде текста + quality_score + route.

## Ограничения

- Система не хранит личный профиль между разными сессиями (в текущей версии)
- Tutor работает через облачный API (Kimi) — требует интернет
- Planner и Evaluator — полностью локальные модели
