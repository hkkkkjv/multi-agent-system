# Skills Reference — Инструменты агентов

## Planner Agent Skills

### `analyze_complexity(topic: str) -> dict`
Оценивает сложность темы по эвристике.
- Вход: название темы
- Выход: `{"topic": str, "complexity": float, "recommended_minutes": int}`
- Сложные темы (0.8): квантовая физика, дифференциальные уравнения, нейросети
- Обычные темы (0.5): история, литература, базовое программирование

### `build_timeline(topics_json: str) -> list`
Строит план дня из списка тем с дедлайнами.
- Вход: JSON-строка `[{"name": str, "deadline": str, "minutes": int}]`
- Выход: список блоков с временем, темой, длительностью
- Сортирует по срочности: "завтра" идёт первым

### `microtask(topic: str, duration_min: int = 10) -> dict`
Генерирует короткую задачу для восстановления фокуса.
- Вход: тема, длительность в минутах
- Выход: `{"topic": str, "duration_min": int, "task": str}`
- Используется когда пользователь застрял или устал

### `adjust_plan_after_miss(topics_json: str, missed_topic: str) -> list`
Перераспределяет материал после пропущенного дня.
- Сокращает пропущенную тему до 70% от исходного времени
- Добавляет пометку о необходимости сфокусироваться на ключевых концепциях

## Tutor Agent Skills

### `explain_topic(topic: str, level: str) -> str`
Объяснение темы с адаптацией под уровень.
- Уровни: `beginner` / `intermediate` / `advanced`
- beginner: примеры из жизни, без терминов
- intermediate: термины с объяснениями
- advanced: точные технические детали

### `find_analogies(topic1: str, topic2: str) -> str`
Находит 2-3 аналогии между двумя темами.
- Используется для связывания нового материала с уже известным
- Например: JOIN в SQL ↔ пересечение множеств в математике

### `make_summary(text: str) -> str`
Структурированный конспект.
- Формат: Ключевые идеи / Термины / Что важно запомнить

### `rag_search(query: str) -> list`
Поиск по загруженным пользователем документам (ChromaDB).
- Возвращает top-3 релевантных отрывка с источником
- Используется когда пользователь загружал конспекты/PDF

## Evaluator Skills

### `quality_score() -> float`
Итоговая оценка 0.0–1.0.
- >= 0.8: ответ принят
- < 0.8: retry (максимум 2 раза)
